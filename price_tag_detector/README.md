# Price Tag Detector 🏷️

Автоматическое обнаружение, отслеживание и извлечение ценников из видео.

## Что делает проект

| Шаг | Описание |
|-----|----------|
| 1 | Детектирует ценники на каждом кадре с помощью **YOLOv8n** (самая лёгкая и быстрая версия) |
| 2 | Отслеживает каждый ценник во времени (встроенный ByteTracker) |
| 3 | Для каждого трека выбирает кадр с **максимальной резкостью** (метрика Лапласиана) |
| 4 | Сохраняет вырезанный ценник как `.jpg` файл |
| 5 | Записывает координаты и время в `output/results.csv` |

## Установка

### Шаг 1 — Установить Python 3.10 или 3.11

Скачать: https://www.python.org/downloads/

> ⚠️ При установке обязательно поставить галочку **"Add Python to PATH"**

### Шаг 2 — Установить зависимости

Запустить `install.bat` (двойной клик).

Скрипт автоматически:
- Установит PyTorch 2.3 с поддержкой CUDA 12.1
- Установит YOLOv8 (ultralytics)
- Установит OpenCV
- Скачает веса модели `yolov8n.pt` (~6 МБ)

## Использование

### Обработка видео

1. Скопировать видеофайлы (`.mp4`, `.avi`, `.mov`, `.mkv`) в папку `input_videos\`
2. Запустить `run.bat`

### Результаты

```
output\
├── crops\
│   ├── video1__a3f9bc12__1250ms.jpg    ← ценник, уникальный id + время в мс
│   ├── video1__d7e21a44__3870ms.jpg
│   └── ...
└── results.csv                          ← все данные в таблице
```

### Формат CSV

```csv
video,unique_id,tracker_id,timestamp_ms,x,y,w,h,sharpness,crop_file
video1,a3f9bc12,1,1250,142,88,210,95,312.4,video1__a3f9bc12__1250ms.jpg
```

| Поле | Описание |
|------|----------|
| `video` | Имя видеофайла (без расширения) |
| `unique_id` | Уникальный 8-символьный ID ценника |
| `tracker_id` | Внутренний ID трекера для данного видео |
| `timestamp_ms` | Время лучшего кадра от начала видео (мс) |
| `x, y, w, h` | Координаты bbox в пикселях |
| `sharpness` | Значение резкости (больше = чётче) |
| `crop_file` | Имя файла с вырезанным изображением |

## Параметры командной строки

```
python src\detector.py --help

  --videos    путь(и) к видео (поддерживает glob: "input_videos\*.mp4")
  --model     путь к модели (default: models\yolov8n.pt)
  --output    папка для результатов (default: output)
  --conf      порог уверенности детекции 0..1 (default: 0.35)
  --iou       порог NMS IoU (default: 0.45)
  --max-missed  кадров до удаления трека (default: 30)
  --cpu       использовать CPU вместо GPU
```

### Настройка под задачу

| Ситуация | Рекомендация |
|----------|-------------|
| Много ложных срабатываний | Увеличить `--conf` до 0.5–0.6 |
| Пропускает мелкие ценники | Уменьшить `--conf` до 0.25 |
| Трек обрывается на быстрой камере | Увеличить `--max-missed` до 60 |
| Нет GPU | Добавить флаг `--cpu` |

## Как работает детектор

```
Видео
  │
  ▼
YOLOv8n (детекция ценников)
  │
  ▼
ByteTracker (присваивание ID, отслеживание между кадрами)
  │
  ▼
Для каждого трека: вычислить резкость Лапласиана
  │
  ▼
Выбрать кадр с максимальной резкостью
  │
  ├─► Сохранить crops/videoname__uid__Xms.jpg
  └─► Записать строку в results.csv
```

**Метрика резкости**: дисперсия оператора Лапласа на изображении в оттенках серого.
Чем выше значение — тем чётче изображение (меньше размытия).

## Примечание о модели

По умолчанию используется `yolov8n` (nano) — самая лёгкая версия:
- ~6 МБ весов
- ~4ms на кадр на RTX 4060
- Обнаруживает объекты общего назначения (класс `book`, `cell phone` и др.)

> 💡 **Для лучшей точности**: если у вас есть датасет ценников, можно дообучить модель командой  
> `yolo train model=yolov8n.pt data=price_tags.yaml epochs=50`
